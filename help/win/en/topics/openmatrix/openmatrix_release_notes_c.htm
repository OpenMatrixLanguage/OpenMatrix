Welcome to OpenMatrix

OpenMatrix is an open source project. To learn more about the OpenMatrix project, please visit https://www.openmatrix.org/.

OpenMatrix 1.0.8 includes the following new features and enhancements.

OML Commands and Functions

  • New OML profiler functions.
  • New removehiddenpath command.
  • The -ascii option of save and dlmwrite can now use the same format.
  • When outputlog is enabled and a script is executed, any errors are recorded in the output log.
  • The EndOfline option is implemented for the textscan command.
  • The system command in async mode is enhanced to work on all cores, on Linux. Performance is also improved.
  • The eval function is optimized for simple numeric expressions.
  • The dlmwrite command is backward compatible with earlier OpenMatrix versions.
  • The xlsopen and xlswrite documentation explains that a new file is created if no file exists.
  • Support for a single argument (y), in addition to (x,y), has been added for cumtrapz:
    
area = cumtrapz(y)
area = cumtrapz(x,y)

Resolved Issues

The following issues are resolved:
  • Running a script with outputlog produces an empty TXT file.
  • CSV file-reading errors occur due to line ending ASCII characters.
  • textread crashes OML.
  • eval(['if 1',char(10),'expr=3;',char(10),'end']) should not have a return value.
  • ls doesn't work correctly with a wildcard on Linux.
  • Crash on vectorized assignment of variable.
  • Crash when str2func is called due to text within quotes.
  • isempty not working for class objects..
  • Unable to find a function depending on its position within a file.
  • Issue with fit quality when applying the polyfit function with a high polynomial order.
  • freqz documentation does not mention the 2nd output value.
  • Matrix power calculation bug with inappropriate diagonalization.
